变量约定

_foo 单下划线，代表不能直接访问的类属性，需要通过类提供的接口进行访问，不能用from xxx import导入。
foo 双下划线代表类私有成员。 foo 前后双下划线的，代表python的特殊方法，如init__()

编码风格

  1. 使用4个空格来缩进，不要使用tab，更不能混用。
  2. 函数与函数间隔一行
  3. 类与类间隔两行
  4. 少使用inline comments(行内注释)
  5. 尽量使用block comments（用#开头的多行函数注释），保持与目标函数同缩进
  6. docstring，开头一行写返回值，接下来一行写函数功能描述。

命名约定

  1. 尽量少用’i’,’I’,’O’单字母作为命名
  2. 包和模块应该用短且全小写字符：thismoduleisnew
  3. class name则建议用CapWords（首字母大写）的命名方式，NewClassName
  4. 全局变量尽量只是设计用于模块内使用，并且可以用import *时，包内的all机制来除去全局变量。
  5. 函数名(function name)尽量都用小写且单词间以_（下划线）连接，另外，大小混合的mixedCase值允许用于传统类中。
  6. 常量建议全大写，单词间以’_’间隔。：THIS_IS_CONSTANT_VAR
